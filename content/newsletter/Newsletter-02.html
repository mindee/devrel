Invoices v2.1

Released during the summer, this new minor version release of our Invoices API brings us closer to our goal: human-level accuracy. You have nothing to do on your end to benefit from this release, the models behind the APIs were all improved but there was no change in the API itself. The main improvements:
  • Extraction accuracy improved for every field
  • Better geographic robustness
  • New currencies supported: PLN, HUF, NOK, XPF, CZK, DKK, GBP, SEK, CHF, EUR, USD, BRL, AED, AUD, SGD, CAD, COP, HKD, NZD, USD, GNF, JPY, CNY
  • Better document-level predictions resulting in higher accuracy on multi-page pdf invoices 

We've also added lots of new data to our test data set to make sure we were shipping a product robust on all your possible data sets out there. We benchmarked our API on 4,000 randomly selected invoices, each of them annotated three times by humans. In order to fit as well as possible a production distribution, those test invoices come from 60 different countries (FR, US, UK, DE, ES, IT, NL represents 83% of the data set), 33% of them are multi-pages. Doctr: Open Sourcing a general-purpose OCR 
We've recently released an open-source general-purpose OCR based on Tensorflow2 and Pytorch: Doctr - Document Text Recognition

Why?

🤓 When we started working on our document understanding technology, we couldn't find any non-commercial high-performance OCR. It's still true today, you don't have many reliable solutions if you don't want to use an API, let alone if you don't want to pay for it.

🎯 We want to provide every software developer with the right tools to process documents in their applications. While our commercial APIs are mainly focused on automating key information extraction, we felt that OCR was a relevant addition to our toolkit to help developers on other use cases such as image-to-text or translation for instance.
